# Spring Boot 多环境配置说明

## 📋 配置文件说明

本项目已配置多环境支持，包含以下配置文件：

1. **application.yml** - 主配置文件（通用配置 + 默认激活环境）
2. **application-dev.yml** - 本地开发环境配置
3. **application-prod.yml** - 生产环境配置

## 🔧 环境切换方法

### 方法1：修改主配置文件（推荐用于开发）

在 `application.yml` 中修改 `spring.profiles.active` 的值：

```yaml
spring:
  profiles:
    active: dev   # 本地开发环境
    # active: prod  # 生产环境（注释掉dev，取消注释prod）
```

### 方法2：启动参数（推荐用于生产）

#### IDEA/Eclipse 开发环境
1. 右键 `HealthineersVisualizationApplication.java`
2. 选择 `Run` -> `Edit Configurations...`
3. 在 `VM options` 中添加：
   ```
   -Dspring.profiles.active=dev
   ```
   或
   ```
   -Dspring.profiles.active=prod
   ```

#### 命令行启动（JAR包）
```bash
# 本地开发环境
java -jar healthineers-visualization.jar --spring.profiles.active=dev

# 生产环境
java -jar healthineers-visualization.jar --spring.profiles.active=prod
```

#### Maven 打包时指定环境
```bash
# 打包时指定环境
mvn clean package -Pdev
# 或
mvn clean package -Pprod
```

### 方法3：环境变量（推荐用于服务器部署）

#### Linux/Mac
```bash
export SPRING_PROFILES_ACTIVE=prod
java -jar healthineers-visualization.jar
```

#### Windows
```cmd
set SPRING_PROFILES_ACTIVE=prod
java -jar healthineers-visualization.jar
```

#### 宝塔面板（推荐）
在宝塔面板的 Java 项目管理中：
1. 进入项目设置
2. 在 `JVM参数` 或 `启动参数` 中添加：
   ```
   --spring.profiles.active=prod
   ```

- **prod**: 允许 `47.98.159.249:8001` 访问

## 🚀 部署到生产环境步骤

### 1. 修改生产环境配置

编辑 `application-prod.yml`，根据实际服务器环境修改：

```yaml
# 数据库配置（如果用户名密码不同）
spring:
  datasource:
    username: your_prod_username
    password: your_prod_password

# Python路径（确认服务器Python路径）
python:
  executable: /usr/bin/python3  # 或 /usr/local/bin/python3

# 项目路径（根据实际部署路径修改）
python:
  project-root: /www/server/healthineers
  backend-dir: /www/server/healthineers/back-end/healthineers-visualization
```

### 2. 打包项目

```bash
mvn clean package
```

### 3. 上传到服务器

将生成的 `target/healthineers-visualization-0.0.1-SNAPSHOT.jar` 上传到服务器。

### 4. 启动项目（使用生产环境配置）

```bash
java -jar healthineers-visualization-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
```

### 5. 宝塔面板配置

1. 在宝塔面板中创建 Java 项目
2. 上传 JAR 文件
3. 在 `启动参数` 中添加：`--spring.profiles.active=prod`
4. 设置 `JVM参数`（可选）：
   ```
   -Xms512m -Xmx1024m
   ```

## 🔍 验证环境是否生效

启动项目后，查看日志输出，应该能看到：

```
The following profiles are active: dev
```
或
```
The following profiles are active: prod
```

## ⚠️ 注意事项

1. **Python路径**: 生产环境需要确保Python3已安装，并且路径正确
2. **文件权限**: 生产环境需要确保 `uploads/` 目录有写权限
3. **日志目录**: 生产环境的日志目录需要存在且有写权限
4. **数据库连接**: 生产环境数据库用户名密码需要与实际匹配
5. **CORS配置**: 如果前端域名改变，需要更新 `cors.allowed-origins`

## 📞 常见问题

### Q: 如何确认当前使用的是哪个环境？
A: 查看启动日志，会显示 `The following profiles are active: xxx`

### Q: 生产环境Python路径找不到怎么办？
A: 使用 `which python3` 或 `whereis python3` 查找Python路径，然后更新 `application-prod.yml`

### Q: 如何同时使用多个环境配置？
A: 使用逗号分隔：`--spring.profiles.active=dev,prod`（不推荐）

### Q: 配置文件中的变量可以引用吗？
A: 可以，使用 `${变量名:默认值}` 格式，例如：`${DB_HOST:localhost}`
