# Spring Boot 多环境切换指南

## 📋 配置文件说明

项目包含以下配置文件：

- **`application.yml`** - 主配置文件（通用配置 + 环境激活配置）
- **`application-dev.yml`** - 本地开发环境配置（Windows）
- **`application-prod.yml`** - 生产环境配置（Linux服务器）

## 🔄 环境切换方法

### 方法1：修改主配置文件（最简单，推荐用于开发）

编辑 `application.yml` 文件，找到以下配置：

```yaml
spring:
  profiles:
    active: dev   # 本地开发环境
```

**切换到本地开发环境：**
```yaml
spring:
  profiles:
    active: dev   # 本地开发环境
```

**切换到生产环境：**
```yaml
spring:
  profiles:
    active: prod  # 生产环境
```

### 方法2：启动参数（推荐用于生产部署）

#### 方式A：命令行启动
```bash
# 本地开发环境
java -jar healthineers-visualization.jar --spring.profiles.active=dev

# 生产环境
java -jar healthineers-visualization.jar --spring.profiles.active=prod
```

#### 方式B：IDEA/Eclipse 开发工具
1. 右键点击 `HealthineersVisualizationApplication.java`
2. 选择 `Run` → `Edit Configurations...`
3. 在 `VM options` 或 `Program arguments` 中添加：
   ```
   --spring.profiles.active=dev
   ```
   或
   ```
   --spring.profiles.active=prod
   ```

#### 方式C：宝塔面板（推荐用于服务器）
1. 登录宝塔面板
2. 进入 `Java项目管理`
3. 找到您的项目
4. 在 `启动参数` 或 `JVM参数` 中添加：
   ```
   --spring.profiles.active=prod
   ```

### 方法3：环境变量

#### Linux/Mac
```bash
export SPRING_PROFILES_ACTIVE=prod
java -jar healthineers-visualization.jar
```

#### Windows
```cmd
set SPRING_PROFILES_ACTIVE=prod
java -jar healthineers-visualization.jar
```

## 🚀 部署到生产环境步骤

### 1. 修改生产环境配置（如需要）

编辑 `application-prod.yml`，根据实际服务器环境修改：

```yaml
# 如果数据库用户名密码不同
spring:
  datasource:
    username: your_prod_username
    password: your_prod_password

# 如果Python路径不同，先查找Python路径
# 在服务器上执行：which python3
python:
  executable: /usr/bin/python3  # 根据实际路径修改

# 如果项目部署路径不同
python:
  project-root: /www/server/healthineers  # 根据实际路径修改
  backend-dir: /www/server/healthineers/back-end/healthineers-visualization
```

### 2. 打包项目
```bash
mvn clean package
```

### 3. 上传JAR文件到服务器

将 `target/healthineers-visualization-0.0.1-SNAPSHOT.jar` 上传到服务器。

### 4. 启动项目（使用生产环境）

**方式A：命令行启动**
```bash
java -jar healthineers-visualization.jar --spring.profiles.active=prod
```

**方式B：宝塔面板**
1. 在宝塔面板中创建Java项目
2. 上传JAR文件
3. 在 `启动参数` 中添加：`--spring.profiles.active=prod`
4. 点击启动

## ⚠️ 注意事项

1. **Python路径**：生产环境需要确保Python3已安装，路径正确
2. **文件权限**：生产环境需要确保 `uploads/` 目录有写权限
3. **日志目录**：生产环境的日志目录需要存在且有写权限
4. **数据库连接**：确保生产环境数据库用户名密码正确
5. **CORS配置**：如果前端域名改变，需要更新 `cors.allowed-origins`

## 🔍 常见问题

### Q1: 如何确认当前使用的是哪个环境？
**A:** 查看启动日志，会显示 `The following profiles are active: xxx`

### Q2: 生产环境Python路径找不到怎么办？
**A:** 在服务器上执行 `which python3` 或 `whereis python3` 查找Python路径，然后更新 `application-prod.yml`

### Q3: 可以同时使用多个环境配置吗？
**A:** 可以，但不推荐。使用逗号分隔：`--spring.profiles.active=dev,prod`

### Q4: 配置文件中的变量可以引用吗？
**A:** 可以，使用 `${变量名:默认值}` 格式，例如：`${DB_HOST:localhost}`

### Q5: 修改配置文件后需要重新编译吗？
**A:** 不需要重新编译，但需要重启应用才能生效
