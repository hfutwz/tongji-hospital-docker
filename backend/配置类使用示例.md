# 配置类使用示例

## 📋 概述

已创建以下配置类，用于从 `application.yml` 中读取配置，替代硬编码：

2. **FileUploadConfig** - 文件上传路径配置
3. **CorsConfig** - CORS跨域配置（已更新）

## 🔧 如何在代码中使用配置类

### 1. 使用 FileUploadConfig

**修改前：**
```java
private static final String UPLOAD_DIR = "uploads/";
```

**修改后：**
```java
@Autowired
private FileUploadConfig fileUploadConfig;

private String saveUploadedFile(MultipartFile file) {
    String uploadRootPath = fileUploadConfig.getUploadRootPath();
    // 使用 uploadRootPath 保存文件
    // ...
}
```

### 2. CORS 配置（已自动更新）

`CorsConfig` 已更新为从配置文件读取，无需修改代码，只需在 `application-dev.yml` 和 `application-prod.yml` 中配置即可。

## 📝 完整示例：修改 FileController

以下是 `FileController.java` 的完整修改示例：

```java
package com.demo.controller;

import com.demo.config.FileUploadConfig;
// ... 其他导入

@RestController
@RequestMapping("/api/file")
public class FileController {
    
    private static final Logger logger = LoggerFactory.getLogger(FileController.class);
    
    @Autowired
    private FileUploadConfig fileUploadConfig;
    
    @Autowired
    private IInjuryRecordService injuryRecordService;
    
    @Autowired
    private ComprehensiveDataImportService comprehensiveDataImportService;
    
    /**
     * 调用Python脚本进行数据导入
     */
    private boolean callPythonImportScript(String excelFilePath) {
        Process process = null;
        
        try {
            // 从配置类读取Python路径
            String pythonScriptPath = pythonConfig.getScriptPath();
            String pythonPath = pythonConfig.getExecutable();
            String projectRoot = pythonConfig.getProjectRoot();
            String backendDir = pythonConfig.getBackendDir();
            
            // 将相对路径转换为绝对路径
            String absoluteExcelPath;
            if (excelFilePath.startsWith("uploads/") || excelFilePath.startsWith("uploads\\")) {
                absoluteExcelPath = backendDir + "/" + excelFilePath.replace("\\", "/");
            } else {
                absoluteExcelPath = excelFilePath;
            }
            
            logger.info("Python可执行文件: {}", pythonPath);
            logger.info("Python脚本路径: {}", pythonScriptPath);
            logger.info("传递给Python脚本的文件路径: {}", absoluteExcelPath);
            
            // 构建命令
            ProcessBuilder processBuilder = new ProcessBuilder(
                pythonPath, 
                pythonScriptPath, 
                absoluteExcelPath
            );
            
            // 设置工作目录
            processBuilder.directory(new File(projectRoot));
            
            // 启动进程
            process = processBuilder.start();
            
            // ... 其余代码保持不变 ...
            
        } catch (Exception e) {
            logger.error("调用Python脚本失败: {}", e.getMessage(), e);
            return false;
        }
    }
    
    /**
     * 测试Python脚本连接
     */
    @GetMapping("/testPython")
    public Result testPythonScript() {
        try {
            // 从配置类读取路径
            String pythonScriptPath = pythonConfig.getScriptPath();
            String pythonPath = pythonConfig.getExecutable();
            
            File scriptFile = new File(pythonScriptPath);
            
            if (!scriptFile.exists()) {
                return Result.fail("Python脚本文件不存在: " + pythonScriptPath);
            }
            
            // 测试Python命令是否可用
            ProcessBuilder testBuilder = new ProcessBuilder(pythonPath, "--version");
            Process testProcess = testBuilder.start();
            int testExitCode = testProcess.waitFor();
            
            if (testExitCode != 0) {
                return Result.fail("Python环境不可用，请检查Python安装");
            }
            
            Map<String, Object> result = new HashMap<>();
            result.put("message", "Python脚本环境正常");
            result.put("scriptPath", pythonScriptPath);
            result.put("pythonPath", pythonPath);
            result.put("scriptExists", scriptFile.exists());
            result.put("scriptSize", scriptFile.length());
            
            return Result.ok(result);
            
        } catch (Exception e) {
            return Result.fail("Python脚本测试失败：" + e.getMessage());
        }
    }
}
```

## ✅ 优势

1. **环境隔离**：不同环境使用不同的配置文件，无需修改代码
2. **易于维护**：配置集中管理，修改方便
3. **避免硬编码**：路径不再硬编码在代码中
4. **部署灵活**：生产环境只需修改配置文件即可

## 🔍 验证配置是否生效

启动项目后，可以在日志中看到配置信息，或调用 `/api/file/testPython` 接口查看当前使用的Python路径。
